# 모두의 마음가짐 — App Store 스크린샷 캡처 가이드

> 5장 + 각 장면 캡션 카피. iPhone 시뮬레이터에서 1번에 완성.

---

## 사이즈 / 업로드 기준

App Store Connect (2026 기준)는 두 가지 사이즈를 인정:

| 디바이스 군 | 해상도 (W×H) | 권장 시뮬레이터 |
|---|---|---|
| **6.9" iPhone (Pro Max)** | 1320 × 2868 | iPhone 16 Pro Max |
| 6.7" iPhone (이전 Pro Max) | 1290 × 2796 | iPhone 15 Pro Max / 14 Pro Max |

**6.9" 한 세트만 업로드해도 충분** — 작은 기기는 Apple이 자동 다운스케일.

---

## 사전 준비 — 데모 데이터 세팅

스크린샷용 더미 문장 8~10개를 시뮬레이터에서 직접 입력. 카테고리 분포:

```
폴더: "오늘의 영감"
  - 우리는 빠르게 흘러가는 것에 익숙해진다. 그래서 천천히 머무는 것이 가장 큰 사치다.
  - 어떤 문장은 한 번 읽고 잊히지 않는다.
  - 글은 사람을 닮고, 사람은 다시 글을 닮는다.

폴더: "책 메모"
  - 좋은 책은 두 번 산다. 한 번은 책장에, 한 번은 마음에.
  - 읽기는 가장 조용한 대화다.
  - 모든 문장은 누군가의 하루를 견딘 결과다.

미분류:
  - 어제 들었던 한 줄.
  - 산책 중 떠오른 생각.
```

스크린샷 시 status bar 정리:
- 시뮬레이터에서 **Features → Toggle In-Call Status Bar**는 사용하지 말고
- `xcrun simctl status_bar` 명령으로 status bar 통일 (마지막 부록 참고)

다크모드 1~2장 / 라이트모드 3~4장으로 섞으면 톤이 풍부.

## 5장 구성 (권장 순서)

### 1. **첫 인상 — 컬렉션 일람** (라이트)

- **화면:** List, 폴더 칩 "전체" 활성, 문장 4~5개 보이게
- **상단 카피 오버레이:**
  ```
  좋은 문장을
  모으다
  ```
- **핵심:** 깨끗한 타이포 + 콘텐츠가 있는 모습 보여주기

### 2. **OCR — 사진 한 장으로** (라이트, 또는 New 진입 후)

- **화면:** New 화면 (`사진 찍기 / 사진첩에서` 버튼 + 그 아래 본문 textarea가 비어있음)
- **상단 카피:**
  ```
  사진 한 장으로
  자동으로 추출
  ```
- **연출:** 책 페이지 또는 스크린샷이 입력된 상태(텍스트가 본문에 들어간 모습)를 다른 버전으로 1장 더 캡처해도 좋음

### 3. **상세 + TTS / 공유** (라이트)

- **화면:** Detail 화면 — 긴 문장 1개 + ▶︎읽기 / ↗︎공유 / 편집 / 삭제 버튼 다 보임
- **상단 카피:**
  ```
  ▶︎ 듣고, ↗︎ 나누고
  ```
- 또는: `걷는 동안에도, 운전 중에도 들으세요`

### 4. **폴더로 정리** (다크)

- **화면:** List, 폴더 칩 "오늘의 영감" 활성 + 해당 폴더 문장만 표시
- **상단 카피:**
  ```
  주제별로
  나만의 폴더
  ```
- 다크모드로 분위기 전환 — 같은 앱이 두 톤 다 좋아 보인다는 인상

### 5. **매일 알림** (라이트)

- **화면:** Settings → "매일 알림" 섹션 + 시간 picker 펼쳐진 상태 (오전 8:30 등)
- **상단 카피:**
  ```
  매일, 한 줄을
  다시 만나는 시간
  ```
- 알림이 이 앱의 "잊혀짐 방지" 핵심 기능 — 마지막 슬라이드의 hook

## 캡션 오버레이 작업

스크린샷 위에 카피를 얹는 방법 (간단순):

**옵션 A — Preview만으로 (가장 빠름)**
1. 시뮬레이터 캡처(⌘S)
2. macOS Preview에서 열기
3. 도구 → 주석(Markup) → 텍스트 추가
4. Pretendard Bold, 60~80pt, accent #5B4FE5 또는 흰색
5. 카피를 상단 25% 영역에 배치

**옵션 B — Figma 템플릿 (가장 좋음)**
1. Figma 무료 계정에서 1320×2868 프레임 5개
2. 각 프레임 위에 시뮬레이터 캡처 + 카피 레이어
3. PNG로 5장 export

**옵션 C — 텍스트 없이 raw 화면만 (가장 빠르지만 약함)**
- 카피 없이 화면 그대로도 심사는 통과. 다만 클릭률↓.
- 첫 출시·트래픽이 적을 땐 옵션 A로도 충분.

## 시뮬레이터 status bar 통일 (선택)

캡처 직전, 시뮬레이터 안에서:

```bash
xcrun simctl status_bar "iPhone 16 Pro Max" override \
  --time "9:41" \
  --dataNetwork "wifi" \
  --wifiMode "active" \
  --wifiBars 3 \
  --cellularMode "active" \
  --cellularBars 4 \
  --batteryState charged \
  --batteryLevel 100
```

(시뮬레이터 이름은 `xcrun simctl list devices` 로 확인. 부팅 후 실행.)

해제:
```bash
xcrun simctl status_bar "iPhone 16 Pro Max" clear
```

## 캡처 명령 정리

시뮬레이터 부팅 후 화면 띄우고:

```bash
# 화면 캡처를 클립보드 + 파일로
xcrun simctl io booted screenshot --type=png ~/Desktop/moeum-1.png
```

5장 캡처 후 macOS Preview/Figma로 카피 합성 → ASC 업로드.

---

## ASC 업로드 위치

App Store Connect → 본 앱 → 1.0 Prepare for Submission →
**iPhone 6.9" Display** 슬롯에 5장 드래그 (10장까지 가능).

다른 디바이스 슬롯(6.5", 5.5" 등)은 비워둬도 됨 — Apple이 자동 매핑.
