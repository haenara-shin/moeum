### 쟁점 1. RN+Expo가 정말 최선인가?
[결론] iOS-first MVP 기준 RN+Expo는 합리적. 단, 위젯 때문에 “순수 Expo” 환상은 버려야 함.  
[근거] Flutter보다 사용자의 React 학습과 맞고 Android 확장도 쉽다. Swift 네이티브는 UX 최고지만 Android 재작성 비용이 큼.

### 쟁점 2. ML Kit Korean 정확도 vs Apple Vision + ML Kit 분리 운영
[결론] iOS MVP는 Apple Vision 우선, Android에서 ML Kit로 분리하는 쪽이 낫다.  
[근거] 한글 스크린샷 OCR은 플랫폼 네이티브 품질과 권한/성능 안정성이 중요하다. 공통 OCR 추상화만 두면 유지보수 가능.

### 쟁점 3. GPT-4o vision 폴백이 정말 필요한가?
[결론] MVP 필수 아님. “OCR 실패 시 수동 보정”이 먼저다.  
[근거] 비용, 개인정보, 네트워크 의존성이 생긴다. 책/이미지 난독 케이스가 검증된 뒤 프리미엄 기능으로 붙여도 늦지 않다.

### 쟁점 4. TTS 하이브리드 vs 단일 선택
[결론] MVP는 expo-speech 단일. OpenAI TTS는 나중에 유료 기능.  
[근거] 문장 앱에서 핵심은 저장·회상·위젯이다. 고품질 음성은 차별화 요소지만 초기 복잡도를 크게 올린다.

### 쟁점 5. MVP 5개 기능 적정성
[결론] 과하다. 입력/OCR/위젯/알림까지가 1차, 전체 컬렉션 TTS는 후순위.  
[근거] 위젯과 OCR만으로도 네이티브 연결 난도가 있다. TTS 플레이어까지 넣으면 일정 리스크가 커진다.

### 쟁점 6. 4~6주 현실성
[결론] 축소하면 가능, 원안 그대로는 빠듯하다.  
[근거] RN 앱 + SQLite + iOS 위젯 + 알림 + 기본 OCR까지는 가능권. Android, GPT 폴백, 프리미엄 TTS는 2차로 빼야 한다.
