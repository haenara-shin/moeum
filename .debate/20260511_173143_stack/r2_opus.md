# Round 2 — Claude Opus 4.7 Rebuttal

## 1. 다른 의견과 강하게 다른 점
**일정**: 나만 8~10주를 정직한 추정으로 봄. 나머지 3명은 "축소시 6주 가능" 쪽. 
→ 재고하면: 사용자가 React/TS "학습 중"이라고 했지 "능숙"이 아니다. **iOS 앱스토어 첫 제출자라는 변수도 빠졌다.** 첫 제출은 인증·번들 ID·심사 거절·스크린샷·개인정보 정책 작성 등 코드 외 1~2주 소요. 따라서 4개 축소 + iOS only도 **7~8주가 정직**이라는 입장 유지.

**MVP 기능**: 나만 "태그/검색"을 추가하자고 했음. Codex/Sonnet은 4개로 줄이자고 함.
→ 양보: "검색"은 MVP 제외해도 OK(컬렉션 작을 땐 불필요). 단 **"태그 또는 카테고리"는 위젯 필터링 때문에 필수**다. 위젯에서 "오늘은 위로 카테고리만 보여줘" 같은 사용성이 핵심 차별화.

## 2. 다른 의견 중 보강해야 할 점
- **Gemini의 "위젯/알림 디자인 품질에 몰입"** — 동의. 이 앱은 "수집보다 노출"이 본질. 위젯 타이포그래피 4~6종 + 색 팔레트 6종 정도는 MVP에 포함해야 위젯 시장에서 살아남음. 이걸 놓쳤었다.
- **Codex의 "수동 보정 UI가 폴백보다 ROI 높다"** — 매우 동의. OCR 결과를 한 화면에서 빠르게 수정/분할/병합할 수 있는 에디터가 핵심. 폴백 API 키 관리보다 백배 가치.
- **Sonnet의 "Audio Session/백그라운드 재생 함정"** — 동의. TTS는 단순해 보여도 잠금화면 컨트롤·인터럽션 처리·이어폰 제거 시 일시정지 등 표준 동작 구현이 만만치 않음. MVP에서 빼는 게 맞다.

## 3. 모두가 놓친 리스크 (★중요)
- **위젯 데이터 동기화**: iOS WidgetKit은 메인 앱과 별도 프로세스. App Group + 공유 SQLite(또는 UserDefaults/JSON 파일)로 전달해야 하고, 타임라인 갱신은 OS가 통제(주당 ~40~70회 제한). "랜덤 새 문장" 효과를 위해 미리 N개를 큐로 만들어둬야 함.
- **한글 폰트 라이선스/렌더링**: 시스템 폰트(Apple SD 산돌고딕) 외 커스텀 폰트(예: 나눔, 본명조)를 위젯에 쓰려면 라이선스 확인 + 위젯 번들에 폰트 임베드 필요. 일부 한글 폰트는 위젯 사이즈(systemSmall ~158pt)에서 가독성이 무너짐.
- **앱스토어 심사 함정**: "동기부여 명언"류 앱은 심사에서 **"콘텐츠가 너무 적다(Guideline 4.2)"** 거절이 흔하다. 초기 시드 데이터(공유 가능한 무저작권 명언 100~300개) 필수.
- **개인정보 정책**: OCR 이미지 처리는 온디바이스라 데이터 송신 없지만, 정책 문서에는 명시 필요. Privacy Manifest(iOS 17+) 항목 작성 필수.
- **백업/복원**: 1년 모은 문장이 기기 변경시 날아가면 즉시 별점 1점. iCloud Backup으로는 부족, **사용자 트리거 export(JSON/CSV)** 가 MVP에 들어가야.
